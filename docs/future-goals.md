## FG-001 强化 Phaser 像素场景表现

- 状态：等待用户指令
- 优先级：高
- 来源：像素场景基础修复完成后的后续增强建议

### 目标

在不破坏现有架构边界的前提下，进一步增强主场景的像素冒险观感，让中间主玩法区域更接近真正的 2D 像素 RPG 演示效果，而不是只具备基础可运行能力的占位场景。

### 计划方向

- 增强地表与装饰分层表现，让不同区域的地面、路径、障碍与装饰更有像素辨识度
- 提升玩家移动反馈，包括脚下阴影、移动节奏、朝向切换与更稳定的行走表现
- 增强人物与交互点的在场感，例如头顶提示、聚焦高亮与临近交互反馈
- 强化场景氛围元素，例如更克制但更稳定的环境特效与区域特色视觉提示
- 为后续真实美术资源替换保留清晰接口，不把表现逻辑耦合进规则、状态、控制器或持久化层

### 约束

- 仅在渲染层与渲染适配层内推进
- 不把业务规则搬进 Phaser 场景
- 不改变现有保存 / 加载契约
- 不移除当前可测试、可注入、可降级的调试路径

### 执行条件

- 只有当用户明确要求开始该项时，才进入实现阶段

## FG-002 场景内敌人显式可视化

- 状态：等待用户指令
- 优先级：高
- 来源：本轮战斗修复后的下一步建议

### 目标

在地图探索与战斗入口之间建立更直观的视觉连接，让玩家在进入战斗前就能清楚知道“正在和谁战斗”，从而提升演示可读性，减少仅靠右侧敌情列表或底部战斗面板理解战局的负担。

### 计划方向

- 在 `GameSceneViewport` 与 Phaser 场景中为可战斗目标增加显式敌人标记
- 为敌方单位增加名称、小型生命条、危险等级或首领标识
- 让探索信号、敌人实体、战斗入口三者保持一致映射，避免 UI 与规则脱节
- 在首领区域中突出当前可交战目标，减少“地图上看不到敌人”的问题
- 保持与现有 `combat encounter`、`exploration signal`、`review payload` 契约对齐，必要时只扩展渲染所需的视图模型字段

### 约束

- 首领逻辑、战斗合法性与阶段切换仍保留在规则层
- UI 只读取视图模型或控制器暴露的数据，不直接拼接战斗规则
- 不破坏当前调试入口、固定种子与强制战术 / 阶段测试能力
- 所有新增玩家可见文案继续保持简体中文

### 执行条件

- 只有当用户明确要求开始该项时，才进入实现阶段
